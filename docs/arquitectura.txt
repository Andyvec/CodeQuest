La idea es separar el proyecto en 3 capas. Con independencia entre el backend y el frontend.

1 Arquitectura General

1.1 - Fontend
    1.1.1 Lógica de UI en ReactJs (Usado con ES6 y JSX) y MobX
    1.1.2 SemanticUI como framework de interfaz gráfica (Integrado con React)
    1.1.3 Lógica de negocio y datos enteramente en el cliente
    1.1.4 Debe estar desacoplado del backend, conectándose mediante APIs Restfull puras

1.2 - Backend/Servicios
    1.2.1 TECNOLOGÍA A DEFINIR. NodeJs? Java? PHP?
    1.2.2 Servidor estúpidamente simple, conteniendo servicio de sesiones (o tokens) y servicios de la aplicación
    1.2.3 Toda la comunicación con frontend mediante Web Services RESTful puro
    1.2.4 Necesitamos alguna forma para sincronizar el trabajo entre distintos usuarios (WebSockets?)
    1.2.5 La nomenclatura del servicio deberá ser genérica y no atender a los casos particulares del frontend
